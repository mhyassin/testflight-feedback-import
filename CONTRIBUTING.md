# Contributing

## Prerequisites

- Node.js 24+

## Setup

```bash
npm install
```

## Building

The action ships a pre-compiled bundle at `dist/index.js` (generated by [esbuild](https://esbuild.github.io)). You must rebuild it whenever you change source files.

```bash
npm run build
```

This runs `ncc build src/index.ts -o dist` and produces:

- `dist/index.js` — single-file bundle used by the action runner
- `dist/index.js.map` — source map (not committed, listed in `.gitignore`)

**Always commit the updated `dist/index.js` alongside your source changes.** The action runner executes `dist/index.js` directly — it does not install dependencies or compile TypeScript at runtime.

## Project structure

```
src/
  index.ts                                        # Entry point — run() orchestrates the full import flow
  types.ts                                        # TypeScript interfaces for App Store Connect API shapes
  utils/
    appstoreconnect/
      generateAscToken.ts                         # Signs a short-lived ES256 JWT for ASC auth
      fetchAsc.ts                                 # Typed HTTPS wrapper for the ASC REST API
      fetchVersionStrings.ts                      # Fetches version strings for a set of build IDs
    github/
      issueExistsForFeedback.ts                   # Checks for an existing issue by feedback marker
      ensureLabelExists.ts                        # Creates a label if it does not already exist
    format/
      feedbackBody.ts                             # Builds the markdown body for a new issue
      buildIncludedMap.ts                         # Indexes an ASC included array by resource ID
      testerDisplayName.ts                        # Formats a tester's display name
      buildVersion.ts                             # Formats version string (build number)
action.yml                                        # Action metadata: inputs, outputs, entry point
dist/
  index.js                                        # Compiled bundle (committed)
```

Each utility file exports exactly one function. Add a new file rather than adding a second export to an existing one.

## Testing locally

Run the compiled bundle directly by setting GitHub Actions inputs as `INPUT_<UPPERCASED_NAME>` env vars:

```bash
INPUT_ASC_ISSUER_ID="..." \
INPUT_ASC_KEY_ID="..." \
INPUT_ASC_PRIVATE_KEY="$(cat /path/to/key.p8)" \
INPUT_ASC_BUNDLE_ID="com.example.app" \
INPUT_GITHUB_TOKEN="ghp_..." \
INPUT_DRY_RUN="true" \
INPUT_DAYS_BACK="7" \
INPUT_LABEL="testflight-feedback" \
GITHUB_REPOSITORY="owner/repo" \
node dist/index.js
```

Use `INPUT_DRY_RUN="true"` to log what would be created without touching GitHub Issues. Note: deduplication still runs in dry run mode, so the token needs `Issues: Read` permission even for a dry run. For a fine-grained PAT, enable **Issues: Read** (at minimum) for the target repository.

## Making changes

1. Edit the relevant file under `src/utils/` or add a new single-function file if needed.
2. Run `npm run build` to compile.
3. Test locally with `INPUT_DRY_RUN="true"` (see above).
4. Commit all changed source files together with `dist/index.js`.

## Submitting a pull request

- Keep PRs focused — one concern per PR.
- Include a brief description of what changed and why.
- If your change affects inputs, outputs, or behaviour, update `README.md` accordingly.
